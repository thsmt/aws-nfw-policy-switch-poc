# セキュリティ上の考慮点

## 基本方針

この PoC では、通常運用ポリシーを厳格に保ち、パッチ適用時間だけパッチ運用ポリシーへ切り替えます。

パッチ運用ポリシーは、通常運用より広い通信を許可する可能性があるため、常時利用しない前提です。

## パッチ運用ポリシーの設計

パッチ運用ポリシーでは、OS 更新やミドルウェア更新に必要な宛先だけを許可します。

Cloudflare などの CDN 配下にあるリポジトリでは、接続先が変動する場合があります。そのため、許可対象を洗い出す際は、リポジトリ設定、リダイレクト先、証明書検証先、実際の通信ログを確認します。

必要以上に広い許可を入れる場合は、Maintenance Window の時間内だけ有効になるようにします。

## ログ確認

AWS Network Firewall の Alert ログと Flow ログを確認し、パッチ適用時に発生した通信を把握します。

通信許可を追加する場合は、ログを根拠にして、通常運用ポリシーへ反映するか、パッチ運用ポリシーのみに留めるかを判断します。

## IAM 権限

Lambda には、対象の AWS Network Firewall と 2 つの Firewall Policy に対する最小限の権限を付与します。

このテンプレートでは、`DescribeFirewall` と `AssociateFirewallPolicy` に絞っています。
